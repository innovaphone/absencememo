Element: event

Name event
Type Locally-defined complex type
Nillable no
Abstract no
Documentation Specifies an event handler. The attribute 'block' controls, whether the event handler is blocking or not. A non-blocking event handler covers all of the following elements and their child elements. But only within the parent element, where the event handler was specified.

Non-blocking event handlers will be called pretty much like a function. At the end of a non-blocking event handler, the interpretation resumes one instruction after that, which caused the event.

The <return> element may be making sense within an non-blocking handler.

If a handler can not be found for the element, where the event occurred, the interpreter's calling stack will be unwinded until a handler can be found.
Application Data Example:

<event type="dtmf" block="false">

<pbx-getdtmfdigit out-dtmf="$dtmf" />

<switch var="$dtmf">

<case equal="1">...</case>

<case equal="2">...</case>

<case equal="3">...</case>

</switch>

</event>

<while cond="true">

...

<pbx-prompt url="..."/> <!-- here DTMFs can occur -->

...

</while>

The non-blocking event handler covers the entire while-loop.

Element: exec

Name exec
Type Locally-defined complex type
Nillable no
Abstract no
Documentation Executes an HTTP GET CGI-request with the URL 'url'. The Url is supposed to contain arbitrary script variables. The remote CGI script shall return a short voicemail script comprising <assign> elements. The <exec> element thereby provides the means to send arbitrary variables to e.g. a remote Perl-script. The Perl-script can, in turn, modify arbitrary variables within the voicemail script.
Application Data Example: <assign out="$cgpn" value="00431234567"/> <exec url="http://192.168.0.5/countrybynumber.pl?cgpn=$cgpn"/>

The Perl script will return the following script: <?xml version="1.0" encoding="utf-8" ?> <voicemail xmlns="http://www.innovaphone.com/xsd/voicemail6.xsd"> <assign out="$lang" value="Austria"/> </voicemail>
XML Instance Representation

url="xs:string [1]"/>
Schema Component Representation
<xs:element name="exec">
<xs:complexType>
<xs:sequence/>
<xs:attribute name="url" type="xs:string" use="required"/>
</xs:complexType>
</xs:element>
